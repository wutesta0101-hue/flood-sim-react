# 案例驗證：2014 尼泊爾滿卡村（Mankha）附近 Jure 堰塞壩潰決事件

## 1. 目的

本案例用於驗證 `flood-sim-react` 中的 2D 淺水波方程（SWE）洪水擴散引擎，比對對象為真實發生之堰塞壩事件：2014 年尼泊爾 Sindhupalchok 縣桑科西河（Sunkoshi River）上的 Jure 山崩堰塞壩及其後續潰決。

驗證目標為比較模擬輸出（淹沒範圍、到達時間、下游流量歷線、蓄水位下降）與已發表之事件觀測資料，並依據落差校準地形處理、糙率設定與潰壩機制的模擬方式。

## 2. 模擬引擎概述

### 2.1 控制方程

本引擎採用守恆形式的二維聖維南方程（Saint-Venant equations，即二維淺水波方程,SWE）求解，含底床坡度源項與曼寧摩擦源項：

```
∂h/∂t    + ∂(hu)/∂x           + ∂(hv)/∂y             = 0
∂(hu)/∂t + ∂(hu² + ½gh²)/∂x  + ∂(huv)/∂y            = -g h ∂B/∂x - Cf u|U|
∂(hv)/∂t + ∂(huv)/∂x         + ∂(hv² + ½gh²)/∂y     = -g h ∂B/∂y - Cf v|U|
```

其中 `h` 為水深、`u, v` 為深度平均流速分量、`B` 為底床高程、`g` 為重力加速度、`|U| = √(u²+v²)`、`Cf = g n² / h^(1/3)` 為曼寧摩擦係數。

### 2.2 數值方法

| 項目 | 採用方法 |
|---|---|
| 空間離散 | 有限體積法，cell-centered，結構化笛卡爾網格 |
| 界面通量 | HLL 近似黎曼求解器 |
| 底床坡度處理 | Hydrostatic reconstruction（靜水平衡重建），避免在不平地形上的靜水狀態產生假流速 |
| 摩擦項 | 半隱式曼寧摩擦，於水深趨近於零時仍維持數值穩定 |
| 濕乾處理 | 乾格流速歸零，並施以濕乾前鋒流速修正 |
| 時間步進 | 顯式格式，依 CFL 條件限制（Courant 數約 0.4–0.45） |

### 2.3 後處理輸出

後處理模組在模擬過程中累積以下場：

- **到達時間**——每個網格首次變濕的時刻
- **最大水深**——每個網格於整個模擬歷程中的最大水深
- **最大流速**——每個網格於整個模擬歷程中的最大流速
- **破壞參數（Hazard index）**——每個網格 `h·|U|` 的歷程最大值，為潰壩災害評估之標準指標
- **純量統計**——總淹沒面積、全場最大水深、全場最大流速、全場最大破壞參數，以及波前距參考源點的推進距離
- **擴散耗盡判定**——依全場當前最大流速與淹沒面積於一段滑動視窗內的穩定程度所設定之經驗停止條件

### 2.4 原始碼檔案

| 檔案 | 角色 |
|---|---|
| `swe_grid.js` | 網格資料結構，儲存守恆變數 `h`、`hu`、`hv` 與底床高程 `B` |
| `swe_terrain.js` | 地形產生器（目前為合成的山谷+平地地形；本驗證案例將以 DEM 衍生地形取代，見第 5–7 節） |
| `swe_solver.js` | 核心求解器，實作 HLL 通量、hydrostatic reconstruction 與半隱式摩擦更新 |
| `swe_postprocess.js` | 累積災害相關輸出場，並判定擴散耗盡/停止條件 |

### 2.5 設計備註

目前求解器實作的是 Godunov 型有限體積法搭配 HLL 黎曼求解器與 hydrostatic reconstruction，為開發過程中考慮的兩種方案中較嚴謹的一種。另一種省略對流慣性項的簡化局部慣性近似（類似 LISFLOOD-FP 的作法）目前尚未實作，原始碼中已註記其為前端計算效能不足時可替換的低成本方案。

## 3. 事件概述

2014 年 8 月 2 日當地時間約凌晨 2 時 36 分，尼泊爾 Sindhupalchok 縣滿卡村（Mankha）附近的 Jure 村發生一場豪雨誘發之大規模山崩，堵塞桑科西河，形成堰塞壩並在上游造成蓄水。

此事件通常稱為 **Jure landslide**（Jure 山崩）或 **Sunkoshi landslide**（桑科西山崩），台灣的二手報導則多稱其為滿卡村（Mankha）堰塞湖事件，因為 Jure 村位於 Sindhupalchok 縣滿卡村行政區內。

山崩本身造成嚴重破壞與重大人員傷亡，死亡人數普遍報導約為 156 人；後續的潰壩階段因下游疏散與緊急應變已提前啟動，並未造成同等規模的傷亡。

### 4.3 入流量與潰壩出流量

模型校準涉及兩種性質不同的流量指標，分別列示如下。

| 指標 | 通報數值 | 定義 |
|---|---:|---|
| 平均入流量（潰壩前，流入堰塞湖） | 約 155 m³/s | 蓄水期間流入湖泊的平均河川流量（壩體上游測站） |
| 尖峰入流量（潰壩前，流入堰塞湖） | 約 160–198 m³/s | 潰壩前流入湖泊之尖峰河川流量（如 Barhabise 附近測站讀數）；適合作為模型**上游邊界條件**的驗證目標 |
| 尖峰潰壩出流量（潰壩事件期間） | 約 6,436 m³/s | 9 月 7 日潰壩期間，壩體出口處之尖峰流量，依侵蝕型潰壩模型推算；潰壩歷時約 26 分鐘 |

潰壩前入流量（約 155–198 m³/s）與尖峰潰壩出流量（約 6,436 m³/s）應視為兩個獨立、不可互換的校準目標：前者驗證蓄水階段的上游邊界條件，後者驗證模擬中潰壩開口 / 潰決階段的表現。

### 4.4 潰壩與下游影響

壩體曾進行緊急疏洪工程，通報指出於 2014 年 9 月 6 日將疏洪道拓寬約 30 公尺，其後於 9 月 7 日凌晨發生主要潰壩。

通報之潰壩後湖水位下降約 15–18 公尺，下游數公里範圍內有房舍受損，但因預警與疏散已提前完成，潰壩階段並未造成重大人員傷亡。

## 5. DEM 前置處理狀態

### 5.1 原始地形資料

地形來源為地理座標系統（EPSG:4326 / WGS 84）之 SRTM GeoTIFF，網格尺寸為 6000×6000 像元，像元大小為 0.0008333333 度，約對應 3 弧秒解析度。

原始網格 metadata 亦定義 NoData 值為 `-32768`，於匯入模擬流程時應予以保留並明確處理。

### 5.2 用於建模之重投影 DEM

目前處理後的模型 DEM 為投影座標系統 **EPSG:32645 – WGS 84 / UTM 45N** 之 GeoTIFF，座標單位為公尺，適合用於 SWE 網格生成與基於距離的幾何運算。

| 像元大小 | 85.45369563796066 m |
| 網格寬度 | 1226 |

## 6. 座標備註

Jure 事件區域常引用的參考位置約為 **27.767, 85.872**（緯度、經度），但此座標應視為整個堰塞壩區域的概略中心點，而非壩頂的確切位置。

在模型建置時，壩頂、湖泊邊界、潰口斷面與下游驗證斷面，最終應在 QGIS 中以向量幾何明確數化，再重投影至 EPSG:32645，使其與以公尺為單位的 DEM 直接對齊。

## 7. 預定模型設定

處理後的 DEM 將取代 `swe_terrain.js` 中目前的合成地形產生邏輯，使模擬採用實際觀測地形，而非目前理想化的山谷平地地形。

SWE 網格的維度（`nx`、`ny`）與間距（`dx`、`dy`）應依投影後 DEM 之解析度與所需計算成本決定。由於 DEM 已採 UTM 公尺座標，網格間距可直接以公尺定義，不會產生經緯度換算的模糊性。

初始蓄水狀態應依已數化之堰塞湖幾何範圍設定，目標蓄水量約 7×10⁶ m³，壩體高度約 50–55 公尺（見第 4.1 節）；潰壩機制則可採隨時間演變的侵蝕/開口擴大過程，或簡化為分階段開口，並依通報之水位下降、下游反應，以及約 6,436 m³/s 的尖峰潰壩出流量目標進行校準（見第 4.3 節）。

曼寧糙率係數可先採用 `n = 0.03` 作為河谷初始假設值，但應視為待校準參數，而非固定真值。

## 9. 待辦事項

1. 在 QGIS 中確認壩頂與湖泊邊界的確切幾何範圍。
2. 將壩體/湖泊多邊形與中心線或控制斷面，以 EPSG:32645 座標系統匯出為 GeoJSON 或 Shapefile。
3. 將 `DEM/04.tif` 轉換為 React SWE 引擎所需之內部地形表示格式。
4. 以數化之堰塞湖幾何取代目前的合成初始蓄水設定，並採用約 50–55 公尺的壩體高度。
5. 實作或近似一套潰壩開口機制，並依約 6,436 m³/s 的尖峰潰壩出流量目標進行校準。
6. 針對糙率、潰口寬度增長速率、初始湖泊水位進行校準實驗。
7. 比對模擬之淹沒範圍、到達時間與流量歷線行為，與通報之事件特徵進行對照。

## 12. 實作注意事項

- 除非模擬程式碼明確改為其他慣例，否則 DEM 的 NoData 定義應維持為 `-32768`。
- 目前使用的 27.767, 85.872 座標點，應視為區域參考點，而非最終壩頂座標。
- 若需要更小的計算範圍，應以 UTM 座標下的壩體/湖泊多邊形重新裁切投影後 DEM，而非以概略經緯度中心點裁切。
- 地形與初始條件設定中，壩體/擋水高度應採用約 50–55 公尺（見第 4.1 節），而非 100 公尺。
- 潰壩前入流量（約 155–198 m³/s）與尖峰潰壩出流量（約 6,436 m³/s），應維持為兩個獨立、不可互換的校準目標（見第 4.3、8 節）。
